# План работ MediaLib (10 недель)

Документ фиксирует согласованный план проекта на 10 недель и используется как опорный roadmap для реализации, приемки и handoff между агентами.

## Неделя 1 — Инициализация и проектирование

1. Зафиксировать MVP-границы и критерии приемки.
2. Описать доменную модель (`Book`, `AudioBook`, `Video`, `MediaLink`, `Progress`, `User`).
3. Подготовить черновик API-контракта.
4. Подготовить артефакты проектирования (ER/UML/IDEF1x).
5. Подготовить структуру репозитория (`mobile/backend/infra/docs` или эквивалент).

## Неделя 2 — Backend база + Auth

1. Поднять backend-окружение (`venv`, зависимости, запуск FastAPI).
2. Настроить конфиг (`.env`, `config.py`).
3. Подключить PostgreSQL через SQLAlchemy (`db.py`, `health-check`).
4. Создать модель `User`.
5. Подключить Alembic и сделать первую миграцию.
6. Добавить хеширование паролей.
7. Добавить JWT (`access/refresh` + decode).
8. Реализовать `/auth/register`, `/auth/login`, `/auth/refresh`.
9. Реализовать 2FA (`TOTP verify`).
10. Добавить минимальные тесты auth.
11. Обновить документацию недели.
12. Провести приемку auth-флоу.

## Неделя 3 — Каталог и связи

1. Реализовать `media_items` (CRUD).
2. Реализовать `media_links` (связи книга-аудио-видео).
3. Добавить поиск и фильтрацию.
4. Добавить серверную валидацию и ошибки.
5. Написать интеграционные тесты на каталог.

## Неделя 4 — Прогресс и файлы (S3)

1. Реализовать API прогресса (`GET/PUT progress`).
2. Реализовать presigned URL для загрузки/чтения файлов.
3. Реализовать complete-upload и хранение метаданных файла в БД.
4. Добавить базовые ограничения (тип/размер файла).
5. Проверить сквозной сценарий `upload -> play -> save progress`.

## Неделя 5 — Mobile: auth + базовый UI

1. Подготовить клиентскую архитектуру по feature-модулям.
2. Экраны: авторизация, библиотека, профиль.
3. Подключить API auth/catalog.
4. Настроить роутинг и состояния загрузки/ошибок.
5. Включить светлую/темную тему.

## Неделя 6 — Mobile: библиотека, добавление, поиск

1. Экран добавления контента.
2. Экран поиска/фильтрации.
3. Отображение связей между объектами.
4. Корректная обработка пустых и ошибочных состояний.
5. Минимальные widget/integration тесты ключевых экранов.

## Неделя 7 — Плееры и синхронизация прогресса

1. Интегрировать аудио- и видеоплееры.
2. Сохранять/восстанавливать позицию.
3. Периодически отправлять прогресс на сервер.
4. Проверить сценарий продолжения на другом устройстве.
5. Оптимизировать UX воспроизведения.

## Неделя 8 — Офлайн и SQLite

1. Локальный кэш SQLite для каталога/прогресса.
2. Очередь pending sync операций.
3. Retry синка после восстановления сети.
4. Конфликт-стратегия (`lastWriteWins`).
5. Тесты офлайн/онлайн/переключение сети.

## Неделя 9 — Инфраструктура, мониторинг, безопасность

1. Подготовить инфраструктурные конфиги (`IaC/light-IaC`).
2. Настроить TLS и хранение секретов.
3. Настроить ежедневные бэкапы БД.
4. Подключить crash/error monitoring.
5. Добавить метрики/дашборды и CI (`lint/test/build`).

## Неделя 10 — Тестирование, документация, защита

1. Полный прогон функциональных тестов по ТЗ.
2. Нагрузочные и NFR-проверки (API latency, память, старт, размер).
3. Подготовка баг-репортов и финальные фиксы.
4. Подготовка документации (пользователь, админ, методика испытаний, пояснительная).
5. Подготовка и репетиция демо-сценария защиты.

## Принцип выполнения по всем неделям

- Делать инкрементально: фича -> тесты -> документация.
- Поддерживать зеленый тестовый прогон на каждом подэтапе.
- После завершения недели обновлять:
  - статус недели (`docs/weekX_status.md`);
  - приемку недели (`docs/weekX_acceptance.md`, если применимо);
  - `docs/handoff_next_agent.md`.
