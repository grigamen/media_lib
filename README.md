# media_lib

WKR

## Что уже реализовано

### Библиотека и карточки произведений
- Сгруппированный показ форматов одного произведения (книга / аудио / видео).
- Поддержка обложек, описания и жанров в карточках и в детальном экране.
- Добавление, редактирование и добавление нового формата произведения.
- Ограничение на редактирование: кнопка редактирования отображается только владельцу произведения.

### Плеер и просмотр контента
- Раздельный UI для видео и аудио:
  - видео: overlay-управление по тапу, полноэкранный режим, скорость, mute, свайп для выхода из fullscreen (сверху вниз);
  - аудио: отдельный компактный аудиоплеер (не как видеоплеер).
- Прогресс-бар интегрирован в overlay-видео, лишние элементы синхронизации из UI убраны.
- Книги:
  - отображение текстового содержимого в отдельном блоке;
  - поддержка `.txt`, `.md`, `.pdf`, `.epub`, `.docx` для загрузки;
  - корректный парсинг `.docx` (извлечение текста из `word/document.xml`), вместо вывода бинарного мусора.

### Главная и навигация
- На главной:
  - блок `Последнее` вместо `Продолжить чтение`;
  - раздел рекомендаций заменен на `Ранее просмотренные`;
  - удален декоративный/некорректный прогресс в карточке продолжения.
- История просмотренного привязана к аккаунту (для каждого пользователя отдельно).
- Нижняя навигация упрощена:
  - вкладки `Главная`, `Библиотека`, `Профиль`;
  - `Поиск` и `Добавить` открываются как скрытые экраны через действия из интерфейса.
- В `Профиле`:
  - удалены лишние метрики и лишние переключатели;
  - кнопка выхода перенесена в верхнюю часть;
  - кнопка удаления произведений закреплена внизу экрана.

### Backend / права доступа
- Авторизованные пользователи видят в общей выдаче одобренные произведения и **собственные** (в том числе на модерации или отклонённые); модерацию и удаление чужих записей выполняет **администратор**.
- Чтение/просмотр **доступных** пользователю произведений: stream URL файлов и персональный прогресс.
- Изменение и загрузка файлов — для владельца записи; админ имеет расширенные операции удаления.

### Локальный кэш и офлайн (в разработке)
- SQLite: кэш каталога при ошибке сети, локальное зеркало прогресса с отложенной отправкой на сервер, сохранение «недавно просмотренных» на устройстве.
- Юнит-тесты: `test/local_persistence_test.dart`.

### Валидация файлов и типов
- Добавлены проверки совместимости файла с выбранным типом произведения.
- При смене типа в формах добавления/редактирования выбранный файл сбрасывается, чтобы избежать ошибочного сохранения.

## Ограничения / известные моменты
- Встроенное чтение PDF/EPUB как полноценных документов пока не реализовано (для них возвращается сообщение о неподдерживаемом встроенном формате).
- Для запуска backend-тестов требуется установленный `pytest` в Python-окружении; в корне `backend/` ожидается **28** зелёных тестов (`pytest -q`).
- Папка `docs/` с планом недель и handoff для агентов указана в `.gitignore`; чтобы версионировать её в git, добавляй файлы явно (`git add -f docs/...`).
