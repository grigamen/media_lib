# media_lib

WKR

## Что уже реализовано

### Библиотека и карточки произведений
- Сгруппированный показ форматов одного произведения (книга / аудио / видео).
- Поддержка обложек, описания и жанров в карточках и в детальном экране.
- Добавление, редактирование и добавление нового формата произведения.
- Ограничение на редактирование: кнопка редактирования отображается только владельцу произведения.

### Плеер и просмотр контента
- Раздельный UI для видео и аудио:
  - видео: overlay-управление по тапу, полноэкранный режим, скорость, mute, свайп для выхода из fullscreen (сверху вниз);
  - аудио: отдельный компактный аудиоплеер (не как видеоплеер).
- Прогресс-бар интегрирован в overlay-видео, лишние элементы синхронизации из UI убраны.
- **Несколько загруженных файлов** у одного произведения: меню выбора источника («Источник»), переключение presigned URL с сохранением позиции воспроизведения.
- Книги:
  - отображение текстового содержимого в отдельном блоке;
  - поддержка `.txt`, `.md`, `.pdf`, `.epub`, `.docx` для загрузки;
  - корректный парсинг `.docx` (извлечение текста из `word/document.xml`), вместо вывода бинарного мусора.

### Главная и навигация
- На главной:
  - блок `Последнее` вместо `Продолжить чтение`;
  - раздел рекомендаций заменен на `Ранее просмотренные`;
  - удален декоративный/некорректный прогресс в карточке продолжения.
- История просмотренного привязана к аккаунту (для каждого пользователя отдельно).
- Нижняя навигация упрощена:
  - вкладки `Главная`, `Библиотека`, `Профиль`;
  - `Поиск` и `Добавить` открываются как скрытые экраны через действия из интерфейса.
- В `Профиле`:
  - удалены лишние метрики и лишние переключатели;
  - кнопка выхода перенесена в верхнюю часть;
  - кнопка удаления произведений закреплена внизу экрана;
  - для пользователя с правами администратора (`is_admin` в JWT) — вход в **панель администрирования**.

### Администрирование (Flutter + API)
- Две вкладки: **«Подтверждение»** (очередь модерации) и **«Удаление»** (каталог без записей «на модерации»).
- Первая вкладка: только **одобрить** / **отклонить**; вторая: только **удаление** (soft delete на сервере). Записи со статусом `pending` во вторую вкладку не попадают — на клиенте используется параметр API `exclude_pending=true`, на сервере он доступен только администратору.
- Постраничная подгрузка (**«Загрузить ещё»**) для обеих вкладок.
- Уведомления об успехе/ошибке (SnackBar) не копятся в очередь.

### Backend / права доступа
- Авторизованные пользователи видят в общей выдаче одобренные произведения и **собственные** (в том числе на модерации или отклонённые); модерацию и удаление чужих записей выполняет **администратор** (проверка JWT + `User.is_admin` в БД).
- Список произведений: фильтр `moderation_status` (только для админа); для каталога удаления — **`exclude_pending=true`** исключает `pending` из выдачи и из `total`.
- Чтение/просмотр **доступных** пользователю произведений: stream URL файлов и персональный прогресс.
- Изменение и загрузка файлов — для владельца записи; админ имеет расширенные операции удаления и эндпоинты одобрения/отклонения.

### Локальный кэш и офлайн (SQLite)
- **База** (`lib/core/local/media_lib_database.dart`): одна SQLite на устройстве, инициализация при необходимости.
- **Каталог**: `CatalogCacheStore` — несортированный LRU по снимкам на пользователя, ключ кеша по фильтрам поиска; при ошибке сети — **fallback** на «базовый» снимок каталога (с понятным сообщением пользователю).
- **Прогресс**: локальное зеркало + возможный флаг отложенной синхронизации; при появлении сети — догрузка на сервер; слияние с сервером по **LWW** (`lib/core/sync/playback_progress_resolution.dart`, UTC-метки).
- **Связь с сетью**: `connectivity_plus` — при восстановлении соединения выполняется сброс отложенного прогресса.
- **Недавно просмотренные**: персистентность по пользователю.
- Тесты: `test/local_persistence_test.dart`, `test/playback_progress_resolution_test.dart`.

### Валидация файлов и типов
- Добавлены проверки совместимости файла с выбранным типом произведения.
- При смене типа в формах добавления/редактирования выбранный файл сбрасывается, чтобы избежать ошибочного сохранения.

## Сборка и конфигурация

### Клиент → API
- Базовый URL задаётся при сборке: `--dart-define=API_BASE_URL=...` (см. `lib/core/config/app_config.dart`).
- По умолчанию для эмулятора Android: `http://10.0.2.2:8000`.

### Android (Gradle)
- В `android/app/build.gradle.kts` зафиксированы версии `androidx.core`, совместимые с `compileSdk` из шаблона Flutter и AGP проекта (см. комментарий в файле), чтобы сборка не падала на проверке AAR metadata.

## Тесты
- **Flutter**: из корня репозитория — `flutter test`.
- **Backend**: из каталога `backend/` при установленных зависимостях — `pytest -q` (в т.ч. `tests/test_media.py`, `tests/test_auth.py`).

## Ограничения / известные моменты
- Встроенное чтение PDF/EPUB как полноценных документов пока не реализовано (для них возвращается сообщение о неподдерживаемом встроенном формате).
- Папка `docs/` с планом недель и handoff для агентов может быть в `.gitignore`; чтобы версионировать её в git, добавляйте файлы явно (`git add -f docs/...`).
